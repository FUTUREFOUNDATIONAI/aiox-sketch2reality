## Story

**As a** presenter,
**I want** polished loading animations during generation,
**so that** the waiting time itself is part of the show.

## Acceptance Criteria

1. Full-screen overlay with branded gradient background during generation
2. Animated step indicators: "ANALYZING SKETCH LAYOUT" -> "APPLYING FUTURE FOUNDATION BRANDING" -> "GENERATING WEBSITE CODE" -> "DEPLOYING TO VERCEL" -> "WEBSITE IS LIVE"
3. Each step transitions with smooth animation
4. Spinner or pulse animation maintains visual engagement
5. Total animation time adapts to actual pipeline progress (not fixed delays)

## Dependencies

- Story 1.5 (End-to-End Integration & Live Preview) — needs the pipeline with basic loading overlay
- Story 1.2 (Upload Interface with Branded UI) — needs the branded UI foundation

## Dev Notes

**Epic:** Epic 2 — Stage Experience: Polish & Variants

**Technical Context:**
- NFR2: Interface MUST be visually premium and stage-ready — suitable for projection on large screens during live events
- NFR7: Interface MUST handle screen resolutions of 1920x1080 and above for stage projection
- Story 1.5 creates a basic loading overlay; this story enhances it to be stage-spectacular
- All animations must be CSS-based (no external animation libraries needed for MVP)
- Text must be readable from the back of an auditorium on a projected screen

**Branding for Loading Screen:**
- Background: CTA Gradient #aec6ff to #006eeb at 135deg
- Text: Space Grotesk, ALL UPPERCASE, tracking +2%
- Surface containers: glassmorphic with backdrop-blur
- Style: McKinsey meets Silicon Valley — premium, futuristic, jaw-dropping

**Step-to-Pipeline Mapping:**
1. "ANALYZING SKETCH LAYOUT" -> Image uploaded, request sent to Stitch
2. "APPLYING FUTURE FOUNDATION BRANDING" -> Stitch processing with branding prompt
3. "GENERATING WEBSITE CODE" -> HTML extraction from Stitch response
4. "DEPLOYING TO VERCEL" -> Vercel API deployment in progress
5. "WEBSITE IS LIVE" -> Deployment complete, URL available

### Testing

- Visual testing on 1920x1080 resolution
- Verify animations are smooth and professional
- Test with varying pipeline times
- Verify readability from distance (simulating auditorium)
- No unit tests required for MVP
