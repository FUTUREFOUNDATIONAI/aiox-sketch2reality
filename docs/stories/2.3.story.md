## Story

**As a** presenter,
**I want** the result screen to show the deployed URL prominently with a QR code,
**so that** the audience can instantly access the generated website on their phones.

## Acceptance Criteria

1. Deployed URL displayed in large, readable font (visible from back of auditorium)
2. QR code generated automatically from the deployed URL
3. QR code is large enough to scan from audience (minimum 200x200px on screen)
4. Copy URL button with visual confirmation
5. "Open in new tab" button for full-screen preview

## Dependencies

- Story 1.5 (End-to-End Integration & Live Preview) — needs the deployed URL and preview
- Story 1.2 (Upload Interface with Branded UI) — needs the branded UI foundation

## Dev Notes

**Epic:** Epic 2 — Stage Experience: Polish & Variants

**Technical Context:**
- NFR2: Interface MUST be visually premium and stage-ready
- NFR7: Handle screen resolutions of 1920x1080 and above
- The result display replaces/enhances the basic URL display from Story 1.5
- QR code generation can use a lightweight library like `qrcode.js` from CDN or a canvas-based approach
- All elements must be readable from the back of an auditorium when projected

**Layout Considerations:**
- URL and QR code should be positioned where they don't obstruct the iframe preview
- Consider a result "bar" or "card" below or beside the preview
- QR code should have enough quiet zone for reliable scanning
- Both URL display and QR code should use the Horizon Ethos branding

**QR Code Options:**
- CDN library: `qrcode-generator` or similar lightweight option
- Canvas-based generation for zero-dependency approach
- Must work offline once loaded (no API calls for QR generation)

### Testing

- Visual testing on 1920x1080 resolution
- Verify QR code scans correctly from phone
- Test copy-to-clipboard across browsers
- Verify "Open in new tab" works
- Test readability from distance
- No unit tests required for MVP
