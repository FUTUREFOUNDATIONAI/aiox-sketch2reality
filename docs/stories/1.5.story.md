## Story

**As a** presenter,
**I want** the full pipeline connected: upload, generate, extract, deploy, preview,
**so that** one button press does everything.

## Acceptance Criteria

1. GENERATE WEBSITE button triggers complete pipeline
2. Right panel shows live iframe preview of the deployed website
3. Deployed URL is displayed below the preview, copyable to clipboard
4. Loading overlay shows real-time progress steps
5. Error states display clear messages without breaking the interface
6. Pipeline completes in under 90 seconds for a single screen

## Dev Notes

**Epic:** Epic 1 — Core Pipeline: Sketch to Deployed Website

**Technical Context:**
- This is the integration story that connects all previous stories (1.1-1.4) into a single flow
- The pipeline: Upload image -> Call Stitch API -> Extract HTML -> Deploy to Vercel -> Show preview
- NFR1: End-to-end generation time MUST be under 90 seconds for a single screen
- NFR5: System SHOULD gracefully handle Stitch API timeouts or failures with clear user feedback
- The loading overlay here is a basic version; Story 2.2 will enhance it for stage readiness
- Frontend is a single HTML file with inline CSS/JS

**Pipeline Flow:**
1. User uploads sketch image
2. User clicks GENERATE WEBSITE
3. Frontend sends image + settings to `POST /api/generate`
4. Backend: Stitch API generates design from sketch
5. Backend: Extract HTML from Stitch response
6. Backend: Deploy HTML to Vercel
7. Backend: Return deployed URL
8. Frontend: Load URL in iframe, display URL with copy button

### Testing

- Manual end-to-end testing of the complete pipeline
- Test with various sketch images
- Verify timing is under 90 seconds
- Test error scenarios (API timeout, network failure)
- No unit tests required for MVP
